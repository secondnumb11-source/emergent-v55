# ⚖️ بوت ناجز — منصة العدالة (RPA)

بوت أتمتة يعمل بـ **Playwright + Node.js** يفتح متصفح كروم، ينتظر تسجيل دخولك اليدوي
عبر **النفاذ الوطني**، ثم يتنقل آلياً بين صفحات ناجز ويسحب كل البيانات (مع التمرير الكامل
والتعمق في التفاصيل) ويرسلها مباشرة إلى نظامك عبر API الداخلي، ويصدّر نسخة Excel.

## الصفحات التي يغطيها البوت
| الصفحة | الرابط | التعمق | القسم في النظام |
|---|---|---|---|
| القضايا | najiz.sa/applications/lawsuit | ✅ (تفاصيل + أطراف + جلسات + أحكام + طلبات) | إدارة القضايا |
| التقويم العدلي | najiz.sa/applications/dashboard | — | مواعيد الجلسات |
| مواعيد الجلسات | najiz.sa/applications/appointment-requests | — | مواعيد الجلسات |
| الطلبات على القضايا | najiz.sa/applications/lawsuit/requests | ✅ | الطلبات على القضايا |
| طلبات التنفيذ | najiz.sa/applications/iexecution | ✅ | طلبات التنفيذ |
| الوكالات | najiz.sa/applications/wekalat/procurations-query | ✅ | الوكالات القضائية |

الأحكام المسحوبة تنعكس تلقائياً أيضاً في **أرشيف المستندات والأحكام**،
وجلسات القضايا تنعكس في **مواعيد الجلسات** (يتم ذلك على مستوى الخادم).

## التثبيت (مرة واحدة)
يتطلب Node.js 18+ على جهازك:
```bash
cd bot
npm install            # أو yarn install
npx playwright install chromium
cp .env.example .env   # ثم عدّل القيم
```

## الإعداد — ملف `.env`
```
API_BASE=https://your-app-url.com
SYNC_TOKEN=ضع-رمز-المزامنة-من-صفحة-ناجز-في-النظام
MAX_DETAILS=100
```
رمز المزامنة تجده في النظام: **ناجز ← رموز المزامنة ← إنشاء رمز**.

## التشغيل
```bash
node najiz-bot.js
```
أو بديل Puppeteer:
```bash
npm run start:puppeteer
```

> ملاحظة: `start:puppeteer` يستخدم نفس محرّك السحب في `extension/content.js`، لذلك أي تحسين على الإضافة ينعكس على هذا السكريبت.

1. سيفتح المتصفح على صفحة ناجز — **سجّل الدخول عبر النفاذ** (البوت ينتظر تلقائياً حتى 10 دقائق).
2. بعد الدخول يبدأ البوت العمل الآلي: تمرير كامل ↑↓ لكل صفحة، دخول تفاصيل كل قضية/طلب/وكالة،
   النقر على تبويبات اللوحة الجانبية، وإرسال البيانات للنظام **بدفعات كل 5 عناصر** حتى لا يضيع شيء.
3. عند الانتهاء يُحفظ ملف Excel في `bot/output/`.

## خيارات إضافية
- `ONLY_STEPS=cases,executions` لتشغيل صفحات محددة فقط.
- `HEADLESS=1` للتشغيل بدون واجهة (للاختبار فقط — تسجيل الدخول يتطلب واجهة).
- جلسة الدخول تُحفظ في `bot/chrome-profile/` فلن تحتاج نفاذ في كل مرة طالما الجلسة صالحة.

## ملاحظات
- البوت يعيد استخدام نفس محرك السحب الموجود في إضافة كروم (`extension/content.js`)
  فأي تحسين على الإضافة ينعكس على البوت تلقائياً.
- الإرسال يتم عبر `POST /api/public/najiz-sync` مع ترويسة `X-Sync-Token`.
